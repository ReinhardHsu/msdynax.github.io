Dynamics AX 2012 R2 System and Common Properties

撰写于 2015-01-05 修改于 2018-05-01 分类 Microsoft Dynamics AX 标签 Microsoft Dynamics AX

在应用程序对象树（AOT），大多数应用程序对象有一个标准的系统属性的集合。系统属性是只读的。

Microsoft Dynamics AX中，AOT中的任何项目的属性，都可以在在属性窗口查看。要启动属性窗口中，右键单击一个AOT节点，然后单击属性 。在属性窗口中的分类选项卡中，许多系统属性列在统计节点。

本主题列出许多AOT节点上都有的公共属性，但也不是每个AOT节点都有这些。

下表显示了被发现在几乎所有的AOT节点的系统属性。都是只读的。









































属性描述New in this version of

Microsoft Dynamics AX
ChangedBy谁最后修改的对象（通常是发行版本）的用户。
ChangedDate该对象最后修改的日期。
ChangedTime该对象是最后一次改变。
CreatedBy谁创建对象的用户的名称。
CreationDate所创建的对象的日期。
CreationTime已创建对象的时间。

下表显示了在AOT节点中发现的其他常见的属性。




































属性描述New in this version of

Microsoft Dynamics AX
ConfigurationKey指定控制访问或显示元素的配置键。如果用户不能访问配置键，则该元素将不可见。元素包括表单，表单上的控件，表格等元素。AX 2012
LegacyID从旧版本的Microsoft Dynamics AX中指定标识元素。在从以前的版本升级，旧的标识符将被分配到LegacyID。安装特定标识符不会被分配和业务逻辑将保持不变。这不用于新的元素。AX 2012
NeededAccessLevel定义针对用户的最低访问级别。只读。AX 2012
Origin指定一个GUID唯一标识的任何AOT元素。它是由系统分配后的值不会改变。无产地的GUID值的任何地方复制到系统中。这个属性用于在同步和升级方案来标识元素。只读。AX 2012
SecurityKey过时了，但在从Microsoft Dynamics AX的早期版本进行升级的系统保留以供参考
