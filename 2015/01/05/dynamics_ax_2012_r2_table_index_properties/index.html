Dynamics AX 2012 R2 Table Index Properties

Created at 2015-01-05 Updated at 2018-05-01 Category Microsoft Dynamics AX Tag Microsoft Dynamics AX

下表描述了可用于表的索引的属性。























































属性描述New in this version of

Microsoft Dynamics AX
AllowDuplicates如果设置为 Yes，该指数可以是非唯一的。如果你没有创建一个唯一索引，Microsoft Dynamics AX会通过结合第一个索引和RECID来创建一个。
AlternateKey控制这个索引是否是一个替代键的一部分。索引字段必须在每个记录中有一个唯一值。AX 2012
ConfigurationKey设置配置键。如果通过配置键禁用索引字段, 会从索引中自动删除。
Enabled允许您禁用索引。
ID对象内部标识符。
Model指定哪个型号的表索引所在模型是在一个层中元素的逻辑分组。一个元素可以在一个层只有一个模式存在。元素的例子是一个表或类。相同的元件可以在一个模型的定制版本中存在较高的层。AX 2012
Name指定索引名。
ValidTimeStateKey指定此索引键是否被用来确定与附表之间的valid time state关系。默认为No 。












Tip
需要设置AllowDuplicates为No并且AlternateKey为 Yes才能启用该属性。


AX 2012
ValidTimeStateMode指定是否允许两个date effective记录之间的间隙生效。默认值是NoGap。












Tip
需要设置AllowDuplicates为No, AlternateKey为Yes, ValidTimeStateKey为 Yes，使启用该属性。


AX 2012












注
Forms sort on the first index.
